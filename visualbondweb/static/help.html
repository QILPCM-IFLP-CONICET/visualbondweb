Visualbond — spectrojotometer web interface

Determine magnetic coupling
constants from first principles

Visualbond is a web interface for the spectrojotometer library. It implements the method of Matera et al. (Comput. Mater. Sci. 178, 109628, 2020) for the optimal determination of exchange coupling constants Ji in a Heisenberg spin Hamiltonian from ab initio (DFT) total energies computed for selected spin configurations of your magnetic material.

H = − Σ⟨i,j⟩   Jij   Si · Sj
# J > 0 → ferromagnetic  |  J < 0 → antiferromagnetic

→ Workflow overview

The application is organized as a sequential three-step workflow. Each step corresponds to one tab in the interface. Tabs 2 and 3 are unlocked automatically once a model is loaded and validated.

TAB 1 Define Model Load .cif / .struct
set bonds & supercell
TAB 2 Spin Configurations Generate or import spin
configs, inspect equations
TAB 3 Evaluate Enter DFT energies,
compute Ji
1
Load your crystal structure. Upload a .cif or .struct file containing the magnetic atom positions.
2
Add bonds. Define the exchange interaction range by specifying minimum and maximum bond lengths (in Å). Visualbond identifies all symmetry-inequivalent bond types within that range.
3
Generate spin configurations. Use the optimizer to produce a set of spin configurations that, when used as DFT inputs, will give the best-conditioned linear system for extracting J values.
4
Run ab initio calculations. Take the generated spin configurations to your DFT code (VASP, Quantum ESPRESSO, etc.) and compute total energies for each.
5
Enter energies and evaluate. Paste the total energies back into Tab 3, then press Estimate parameters to obtain the coupling constants with error bounds.

⬡ Interface layout

The Visualbond window is divided into four main regions:

RegionDescription
Top bar Application title, File / Edit / Help menus, and a status pill showing the currently loaded file (green dot = model loaded).
Tab bar Three numbered tabs. Tabs 2 and 3 are greyed out until a valid model is loaded and the CIF passes validation on leaving Tab 1.
Main area The active tab's content: sidebar with controls on the left, a central text editor, and (on Tabs 2–3) a results panel on the right.
Status / Log bar A persistent terminal-style log at the bottom of the window. All operations append messages here in real time.

1 Tab 1 — Define Model

This is where you load your crystal structure, set the interaction range, and generate bond types. No calculation is possible until a valid model is defined here.

Loading a model file

Visualbond accepts two file formats:

FormatDescription
.cif Crystallographic Information File — the standard interchange format for crystal structures. Most databases (ICSD, COD, Materials Project) export this format.
.struct WIEN2k / spectrojotometer structure format.

To load a file, use any of these methods:

  • Click the upload zone in the sidebar ("Drop .cif / .struct here") to open a file browser.
  • Use File → Open model file… from the menu bar.
  • Press Ctrl+O.

On successful upload the CIF text of the parsed model is loaded into the editor, and the status bar shows the atom count and cell size.

ℹ If you upload a .struct file, Visualbond converts it to CIF internally. The editor always shows the CIF representation.

Adding bonds

The Add bonds panel defines which pairs of magnetic sites are considered exchange-coupled. Visualbond groups all symmetry-equivalent pairs within the distance range into named bond types (e.g. J1, J2, …).

ParameterDefaultDescription
rmin 0.0 Å Minimum bond length. Bonds shorter than this are ignored. Useful to exclude on-site self-interactions.
rmax 4.9 Å Maximum bond length. All symmetry-inequivalent magnetic pairs within this distance are included. Increase to capture more distant neighbors; be aware that more bonds require more spin configurations.
Discretization 0.02 Å Resolution used to distinguish bond lengths. Two bonds closer than this value are treated as equivalent. Reduce for very dense structures.

Press ⬡ Add bonds to run the bond search. The CIF text in the editor is updated with the new bond information, and the status bar shows the number of bond types found.

⚠ No bonds defined? Tabs 2 and 3 will still be accessible, but the Optimize and Evaluate operations will refuse to run. Always add bonds before proceeding.

Growing the lattice (supercell)

The Grow lattice panel lets you expand the primitive cell into a supercell before bond generation. This is necessary when the spin configurations you need span more than one unit cell.

ParameterDefaultDescription
Lx 1 Repetitions along the a lattice vector.
Ly 1 Repetitions along the b lattice vector.
Lz 1 Repetitions along the c lattice vector.

Press ↗ Apply & grow (or equivalently Add bonds with the supercell values set) to regenerate bonds on the expanded cell.

✦ Tip: A 2×2×1 supercell doubles the number of independent sites and therefore allows more complex (longer-wavelength) spin patterns. Start with 1×1×1 and only enlarge if the optimizer reports a poor condition number.

CIF editor

The central panel is a plain-text editor showing the current CIF content. You can edit it directly — for example to manually correct site labels, adjust occupancies, or add custom bond entries. Changes are validated automatically when you leave Tab 1.

The toolbar at the bottom offers:

  • Open .cif — open a new file (same as the upload zone).
  • Save — download the current CIF as model.cif.
  • A line count indicator on the right.
ℹ When you navigate to Tab 2 or Tab 3, Visualbond silently validates the CIF content. If the file cannot be parsed, the tab switch is blocked and an error is shown in the log.

2 Tab 2 — Spin Configurations & Couplings

Here you build and inspect the set of spin configurations that will be used as DFT inputs. The goal is a set of configurations that gives a well-conditioned linear system for extracting the J values.

Spin configuration file format (.spin)

Configurations are stored in a plain-text format. Each non-comment line represents one spin configuration:

# Spin configurations definition file # Energy [config] # label nan [0,1,0,1,0,1,0] # Antiferro_1 nan [1,0,1,0,1,0,1] # Antiferro_2 nan [0,0,0,0,0,0,0] # Ferro_up nan [1,1,1,1,1,1,1] # Ferro_down
ColumnDescription
Energy Total energy from DFT (in the units your code uses, e.g. eV). Use nan when the energy is not yet known (Tab 2 stage) — these entries are used to build equations but excluded from the fit.
[config] A list of 0s and 1s, one per magnetic site in the cell. 0 = spin up, 1 = spin down.
# label Optional comment used to identify the configuration in plots and output. If omitted, a numeric index is generated automatically.

Optimizing configurations

The Optimize panel generates a set of spin configurations that minimizes the condition number of the linear system A·J = E. A smaller condition number means more reliable J estimates.

ParameterDefaultDescription
Num. configs 10 Number of new configurations to generate and append to the editor. You need at least Nbonds + 1 configurations with known energies to determine all coupling constants.
Bunch size 10 Internal parameter: the size of the candidate pool evaluated at each optimization step. Larger values improve quality at the cost of speed.
Iterations 100 Number of optimization iterations. More iterations yield a better condition number but take longer.

After pressing ⬟ Optimize, new lines are appended to the configuration editor with energies set to nan and a comment showing the condition number estimate.

◈ Optimal independent set — finds the smallest subset of the current configurations that spans the full coupling-constant space. Useful for reducing the number of DFT calculations needed.

Equations panel

The right-hand panel shows the linear equations that relate total energies to coupling constants, derived from the current set of configurations. The output format can be switched between:

FormatDescription
PlainHuman-readable ASCII. Good for a quick check.
LaTeXReady to paste into a paper or thesis.
WolframWolfram Language syntax for use in Mathematica / Wolfram Alpha.

The equations panel updates automatically as you type in the configuration editor (with a short debounce delay). Press ↺ Recalculate to force an immediate update.

✦ Importing from another model: Use ↗ Load configs from other model to map configurations computed for a different (but related) structure onto the current model. A site-mapping string and length tolerance allow you to reuse existing DFT data.

3 Tab 3 — Set Energies & Evaluate

Once you have computed DFT total energies for your spin configurations, come here to enter them and extract the coupling constants.

Entering DFT energies

The central editor in Tab 3 shares the same format as Tab 2. Open your .spin file (or switch from Tab 2 — the content is kept in sync) and replace each nan with the corresponding total energy from your DFT code.

# Spin configurations with DFT energies (eV) -143121.12 [0,0,0,0,0,0,0] # Ferro_up -143121.02 [1,1,0,0,1,1,1] # Disorder_1 -143121.24 [1,0,1,1,1,1,0] # Disorder_2 nan [0,1,0,1,0,1,0] # (no energy yet)
ℹ Lines with nan are included in the equation display but excluded from the fit. You can keep them in the file as a reference.

You need at least Nbonds + 1 lines with numerical energies. If you have fewer, Estimate parameters will report an error and tell you how many more are needed.

Error bound method

Visualbond can estimate error bounds on the J values using two methods:

MethodDescription
Quadratic bound An analytic upper bound derived from the energy tolerance and the condition number. Fast but conservative (overestimates errors).
Monte Carlo Samples random perturbations of the input energies within the specified tolerance and estimates the standard deviation of J. More accurate than the quadratic bound. Recommended for final results.
ParameterDefaultDescription
Energy tol. 0.001 Assumed uncertainty on each DFT energy (in the same units as the energies). This corresponds to your DFT convergence threshold.
Num. samples 1000 (Monte Carlo only) Number of random perturbation samples. 1000 is adequate for most purposes; increase to 5000–10000 for publication-quality uncertainties.
Size factor 1.0 (Monte Carlo only) Scales the amplitude of random perturbations relative to the energy tolerance. Keep at 1.0 unless you have a specific reason to change it.

Reading the results

After pressing ⬗ Estimate parameters, three result panels are populated:

Equations panel

Shows the system of linear equations using only configurations with known energies.

Determined parameters

Each coupling constant appears as a card. The cards show:

  • Name — the bond label (e.g. J1) or E0 for the energy offset.
  • Value — the best-fit coupling constant, in the same energy units as the input energies divided by the number of spins in the cell.
  • Normalized value — the value divided by Jmax, the largest |J| found.
  • Error — the estimated uncertainty (±). A negative error means the system is incompatible (see below).

Card color codes:

Normal   Normal result.    ⚠   Parameter is small relative to the error bound.    ⚠ incompatible   The system is over-determined in a contradictory way (see Errors section).

Energy errors chart

A bar chart showing the normalized residuals ΔEi/ΔE for each configuration used in the fit. These should all lie within ±1 (shown by amber dashed lines) for a compatible model.

Bar colors indicate:

● ≤ 0.7   Well within bounds.   ● ≤ 1.0   Near the bound.   ● > 1.0   Outside bounds — model may be incompatible.

∑ The method

This section gives a self-contained description of the algorithm implemented in spectrojotometer and exposed through Visualbond. For the full derivation and benchmarks see the original publication: doi:10.1016/j.commatsci.2020.109628.

The linear model

The total energy of a magnetic insulator in a collinear spin configuration σ = (σ1, …, σN) ∈ {0,1}N is modelled by a classical Ising/Heisenberg Hamiltonian mapped to the form:

E(σ) = E0 + Σk   Jk   Ak(σ)
# Ak(σ) = number of unlike-spin pairs of bond type k in configuration σ

Here k runs over the symmetry-inequivalent bond types found in the crystal (e.g. first neighbors J1, second neighbors J2, …), and Ak(σ) counts the number of antiparallel spin pairs of that type in configuration σ. E0 is an overall energy offset (the ferromagnetic reference energy).

Computing the total energy for M different configurations gives a linear system:

A · J = E
# A ∈ ℝM×(K+1), J = (J1,…,JK, E0)T, E ∈ ℝM

This system is solved in the least-squares sense when M > K+1. The quality of the determination depends critically on the condition number κ(A): a large condition number amplifies energy errors into large uncertainties on J.

Optimal configurations

The key contribution of the paper is an algorithm to find the set of spin configurations σ(1), …, σ(M) that minimizes the condition number κ(A), thereby requiring the fewest DFT calculations for a given target accuracy.

The algorithm is iterative:

1
Start from any initial set of configurations (or an empty set).
2
At each iteration, generate a bunch of candidate configurations at random and evaluate κ for each candidate added to the current set.
3
Keep the candidate that most reduces κ and add it to the set.
4
Repeat until the desired number of configurations is reached.

This greedy strategy is efficient and produces near-optimal configuration sets in practice. The condition number reported after optimization is an upper bound on |ΔJ| / |ΔE|: the relative error in the coupling constants is at most κ times the relative error in the DFT energies.

ℹ The Optimal independent set button finds the smallest subset of configurations that achieves the same (or similar) condition number — useful when you already have more configurations than strictly necessary and want to minimize DFT runs.

Error bounds on coupling constants

Given DFT energies with uncertainty δE, spectrojotometer provides two kinds of error bounds on the extracted J values:

MethodDescription
Quadratic bound An analytic bound derived from the pseudo-inverse of A:

δJ ≤ ‖A+‖ · δE

Fast to compute and always valid, but typically conservative (overestimates the actual uncertainty). The displayed error is an upper bound, not a standard deviation.
Monte Carlo Samples N random perturbation vectors δE drawn uniformly from [−δE, +δE]M and solves the perturbed system each time. The reported error is the standard deviation of the resulting distribution of J values — a more realistic estimate of the actual uncertainty. Recommended for final published results.

A parameter set is flagged as incompatible when the quadratic bound yields a negative value for one or more Jk — this indicates that the interval [Jk − δJk, Jk + δJk] does not include a self-consistent solution, i.e. the data cannot be explained by the chosen model within the stated energy tolerance.

Plain text, one configuration per line:

# Lines starting with # are comments

⚠ Errors & warnings

Model errors

Cannot parse model: …
The uploaded file could not be read by spectrojotometer. Check that the CIF is valid (try opening it in VESTA or a CIF validator). The error message in the log should indicate the problematic line.
Cannot switch tab: no session. Load a .cif file first.
You attempted to navigate to Tab 2 or 3 before any model was loaded. Open a .cif or .struct file first.
No bonds defined. Run Add bonds first.
The Optimize or Evaluate operations require at least one bond type. Go to Tab 1 and press Add bonds.

Configuration errors

Need at least N configurations with known energies to determine M coupling constants.
You have fewer non-nan energy values than required. Add more DFT results (at least Nbonds + 1) to the configuration editor.
Parse error at line N: expected '<energy> <config>'
A line in the configuration editor is not in the expected format. Check that every non-comment line has an energy (or nan) followed by a bracketed spin list.

Results warnings

⚠ Incompatible model (red parameter cards, energy errors out of bounds)
One or more energy errors lie outside ±1 in the normalized residual chart, and one or more coupling constants have negative error bounds. This means the linear model A·J = E is not consistent with your DFT energies within the stated tolerance. Possible causes:
  • The actual exchange interactions are not well described by a Heisenberg model with only the bond types defined.
  • The DFT energies have larger errors than the tolerance you specified — try increasing Energy tol.
  • Some spin configurations are too similar (linearly dependent) — regenerate with more optimization iterations.
  • You need to include further-neighbor bonds — increase rmax and regenerate.

Bond generation warnings

Bond generation failed: …
An unexpected error occurred in spectrojotometer's bond search. Try adjusting rmin/rmax or the discretization, or check the log for more details.

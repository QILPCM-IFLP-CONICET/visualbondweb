Visualbond — spectrojotometer 0.2

Visualbond extracts the exchange coupling constants Ji of a Heisenberg spin Hamiltonian from ab initio (DFT) total energies:

H = −Σ⟨i,j⟩ Jij Si·Sj
# J > 0 ferromagnetic · J < 0 antiferromagnetic

Workflow

Tab 1 — Load a .cif or .struct crystal structure and define the exchange bond range.

Tab 2 — Generate an optimal set of spin configurations to use as DFT inputs. Inspect the linear equations.

Tab 3 — Enter the DFT total energies and compute the coupling constants with error bounds.

✦ Tabs 2 and 3 unlock automatically once a valid model is loaded and the CIF passes validation on leaving Tab 1.

Tab 1 — Define Model

Load your crystal structure and define which pairs of magnetic sites interact.

Loading a file

Click the upload zone or use File → Open model file… (Ctrl+O). Accepted formats: .cif and .struct. The CIF is shown in the editor and can be edited directly.

ℹ Navigating to Tab 2 triggers a silent CIF validation. If the file cannot be parsed the tab switch is blocked.

Add bonds parameters

ParameterDefaultDescription
rmin0.0 Å Minimum bond length. Pairs closer than this are ignored.
rmax4.9 Å Maximum bond length. All symmetry-inequivalent magnetic pairs within this distance are grouped into bond types (J₁, J₂, …). Increase to include more distant neighbors.
Discretization0.02 Å Resolution for grouping bond lengths. Two bonds closer than this value are treated as equivalent.

Grow lattice (supercell)

Lx / Ly / Lz expand the primitive cell before bond generation. Use a supercell when spin configurations span more than one unit cell.

✦ Start with 1×1×1. Enlarge only if the optimizer reports a poor condition number after generating configurations.

Tab 2 — Spin Configurations

Build the set of spin configurations that minimizes the condition number of the linear system A·J = E.

.spin file format

# Energy   [config]          # label
nan  [0,1,0,1,0,1,0]  # AFM_1
nan  [0,0,0,0,0,0,0]  # Ferro_up

0 = spin up · 1 = spin down · nan = energy not yet known

Optimize parameters

ParameterDefaultDescription
Num. configs10 Configurations to generate. You need at least Nbonds+1 with known energies to determine all J values.
Bunch size10 Candidate pool size per iteration. Larger = better quality, slower.
Iterations100 Optimization steps. More iterations → lower condition number.

◈ Optimal independent set — finds the smallest subset of existing configurations that spans the full coupling-constant space.

Equations panel

Shows the linear system derived from the current configurations. Format can be switched to LaTeX or Wolfram. Updates automatically as you type.

Tab 3 — Set Energies & Evaluate

Replace nan with DFT total energies, then press ⬗ Estimate parameters.

Energy tolerance

The assumed uncertainty on each DFT energy (same units as the energies). Should match your DFT convergence threshold.

Error bound method

MethodDescription
QuadraticAnalytic upper bound — fast, conservative.
Monte CarloSamples random energy perturbations within the tolerance. More realistic. Recommended for final results.

Monte Carlo parameters

ParameterDefaultDescription
Num. samples1000Random perturbation samples. Use 5000–10000 for publication results.
Size factor1.0Scales perturbation amplitude relative to energy tolerance. Keep at 1.0.

Reading results

Parameter cards show the fitted J values ± error. A red / incompatible card means the model cannot explain the data within the stated tolerance — try increasing rmax or the energy tolerance.

The energy error chart shows normalized residuals ΔEi/ΔE. All bars should stay within the ±1 amber lines.

The method

Full derivation in: Matera et al., Comput. Mater. Sci. 178, 109628 (2020). doi:10.1016/j.commatsci.2020.109628

Linear model

The total energy for a collinear spin configuration σ is written as:

E(σ) = E₀ + Σk Jk Ak(σ)
# A_k(σ) = antiparallel pairs of bond type k in σ

Computing M configurations gives the linear system A·J = E, solved by least squares.

Optimal configurations

The optimizer minimizes the condition number κ(A) using a greedy search: at each step it picks the candidate configuration (from a random pool of size bunch) that most reduces κ. A smaller κ means the ratio |ΔJ|/|ΔE| is smaller — more accurate J values for the same DFT precision.

Error bounds

Quadratic: δJ ≤ ‖A⁺‖ · δE — analytic, always valid, conservative.

Monte Carlo: std(J) over N samples of E + Uniform(−δE, +δE) — more realistic estimate.

A negative error bound means the system is incompatible: the data cannot be reproduced by any J within ±δE of the input energies.
